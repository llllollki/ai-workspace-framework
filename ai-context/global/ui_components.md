# UI Components

## Responsibility

This file documents reusable UI component patterns, naming conventions, and usage rules that apply across projects.

## What Belongs Here

- Component naming conventions
- Shared component patterns (forms, cards, lists, modals, etc.)
- Accessibility requirements
- State and variant conventions
- Component composition rules

## What Does Not Belong Here

- Design tokens (see `design_system.md`)
- Application-specific component implementations (belong in the project repo under `components\`)
- One-off page-level layouts

## Related Files

- `design_system.md` — design tokens used by components
- `templates\component_v1.md` — template for documenting a new component
