# ai-workspace-framework

Canonical portable AI context framework for the `c:\Projects\` workspace.

## What This Is

This repository is the single source of truth for the AI working context used across all devices.
It contains the Claude Code entry points (`CLAUDE.md`, `AGENTS.md`) and the `ai-context\` directory
that agents use to orient themselves, load project context, and maintain shared knowledge.

This is **not** an application repo. It contains no application code, environment files, or secrets.

## Canonical vs Mirror

| Location | Role |
|---|---|
| `c:\Projects\ai-workspace-framework\` | **Canonical repo** — edit here, commit here |
| `c:\Projects\CLAUDE.md` | Device-local mirror — generated by sync script |
| `c:\Projects\AGENTS.md` | Device-local mirror — generated by sync script |
| `c:\Projects\ai-context\` | Device-local mirror — generated by sync script |

Never hand-edit the device-local mirror files (except for emergency local notes that will not be
propagated). All reusable framework changes belong in this canonical repo.

## Sync Direction

Sync is **one-way only**:

```
ai-workspace-framework\  →  c:\Projects\  (device mirror)
```

The sync script copies `CLAUDE.md`, `AGENTS.md`, and `ai-context\` from this repo into the workspace
root. It never syncs in the reverse direction. It never deletes destination files.

Always run the dry run first, then apply:

```powershell
# Review what would change
.\setup\sync_framework.ps1 -WhatIf

# Apply
.\setup\sync_framework.ps1
```

## Day-to-Day Workflow

### Making framework changes

1. Edit files in `c:\Projects\ai-workspace-framework\` (e.g., `ai-context\projects\AssistedLivingHelp\`).
2. Commit and push to the private Git remote.
3. Run the sync script to push changes to the device mirror.

### Session changes (AI-written context updates)

During AI sessions, Claude writes to the device mirror (`c:\Projects\ai-context\`). After a session,
copy any framework changes you want to keep back into `ai-workspace-framework\` before committing.
The sync script does not do this automatically — reverse sync is intentionally not supported.

### Picking up framework updates on another device

```powershell
cd C:\Projects\ai-workspace-framework
git pull
.\setup\sync_framework.ps1 -WhatIf   # review
.\setup\sync_framework.ps1            # apply
```

## What Must Never Be Committed

- `AssistedLivingHelp\` — separate application repo with its own Git remote; must never be copied here
- `.claude\` — Claude Code's machine-local memory store; not portable framework content; device-specific
- `.env`, `.env.*` — environment secrets
- `node_modules\`, `.next\`, `dist\`, `build\` — build and dependency output
- Any application source code, config files, deployment files, or generated artifacts

See `.gitignore` for the full exclusion list.
